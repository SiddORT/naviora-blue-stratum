---
name: Alembic autogenerate noise
description: Autogenerate always produces column-comment and index-rename diffs plus a fatal is_active INTEGER cast; must hand-edit before running upgrade.
---

## Rule
Never run `alembic upgrade head` on an autogenerated revision without first inspecting and trimming the file. The only lines that should remain are the `create_table` and their `create_index` calls for the **new** tables.

**Why:** SQLAlchemy column `comment=` attributes cause alembic to emit spurious `alter_column` ops. Existing tables with `Boolean` `is_active` columns trigger an `ALTER COLUMN is_active TYPE INTEGER` that fails with `DatatypeMismatch` in PostgreSQL because there is no implicit cast.

**How to apply:**
1. Run `alembic revision --autogenerate -m "..."` to generate the file.
2. Open the file and delete everything in `upgrade()` except the `op.create_table(...)` and `op.create_index(...)` blocks for the new tables.
3. Mirror the drops in `downgrade()`.
4. Then run `alembic upgrade head`.
