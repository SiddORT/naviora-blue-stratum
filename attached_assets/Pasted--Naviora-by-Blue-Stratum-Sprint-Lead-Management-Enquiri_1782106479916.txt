# Naviora by Blue Stratum

## Sprint – Lead Management, Enquiries & Onboarding Pipeline

Build the Lead Management and Onboarding module.

This module receives all public enquiries and registration requests from:

* Landing Page
* Contact Forms
* Plan Enquiries
* Organization Registration
* Candidate Registration

No accounts should be created directly from public forms.

All requests must go through an approval workflow.

Follow existing platform standards:

* FastAPI
* PostgreSQL
* SQLAlchemy 2.0
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* JWT Authentication
* GDPR Compliance
* Audit Logging
* Encryption Framework

Generate production-ready code.

# Duplicate Prevention

Before creating records:

Check:

* Email
* Phone
* Organization Name

Warn administrators about duplicates.

Do not auto-merge.

---

# GDPR Requirements

Link every enquiry to consent records.

Store:

* Terms Acceptance
* Privacy Acceptance
* Data Processing Consent
* Marketing Consent

Track:

* Consent Version
* IP Address
* User Agent
* Timestamp

Support consent withdrawal.

Never delete consent history.
