A user assigned to an Assessment.

Defines when an Assessment can be accessed.

# Database Design

## assessment_participants

Purpose:

Assign users to assessments.

Fields:

* id

* uuid

* assessment_id

* user_id

* assignment_status

* assigned_at

* assigned_by

* started_at

* completed_at

* attempt_count

* max_attempts_override

* result_status

* remarks

* created_at

* updated_at

* created_by

* updated_by

* deleted_at

* deleted_by

Assignment Status Enum:

* Assigned
* In Progress
* Completed
* Passed
* Failed
* Expired
* Cancelled

Result Status Enum:

* Pending
* Passed
* Failed

Rules:

* One user can only have one active assignment per assessment.
* Prevent duplicate assignments.
* attempt_count starts at 0.
* max_attempts_override is optional.
* If override is null, inherit assessment max_attempts.

Indexes:

* assessment_id
* user_id
* assignment_status

Searchable:

* user_name
* user_email

Filterable:

* assignment_status
* result_status

Sortable:

* assigned_at
* started_at
* completed_at

---

## assessment_schedules

Purpose:

Define assessment availability windows.

Fields:

* id

* uuid

* assessment_id

* schedule_type

* start_date

* end_date

* timezone

* duration_override_minutes

* allow_late_start

* grace_period_minutes

* status

* created_at

* updated_at

* created_by

* updated_by

* deleted_at

* deleted_by

Schedule Type Enum:

* Always Open
* Scheduled Window

Status Enum:

* Draft
* Active
* Archived

Rules:

* Only one active schedule per assessment.
* end_date must be greater than start_date.
* duration_override is optional.
* If null, inherit assessment duration.

---

## assessment_attempts

Purpose:

Track participant assessment attempts.

Fields:

* id

* uuid

* assessment_participant_id

* attempt_number

* started_at

* completed_at

* simulator_session_id

* score

* result_status

* remarks

* created_at

* updated_at

Result Status Enum:

* Pending
* Passed
* Failed
* Abandoned

Rules:

* Attempt numbers increment automatically.
* Maximum attempts enforced.
* One simulator session per attempt.
* Results will be populated in future sprints.

Indexes:

* assessment_participant_id
* simulator_session_id

# Business Rules

When assigning participants:

* Validate assessment status is Active.
* Validate participant is active.
* Prevent duplicate assignments.
* Initialize attempt_count to 0.

When scheduling:

* Validate date ranges.
* Validate timezone.
* Validate duration rules.

When creating attempts:

* Enforce maximum attempts.
* Increment attempt_count automatically.
* Create audit logs.
