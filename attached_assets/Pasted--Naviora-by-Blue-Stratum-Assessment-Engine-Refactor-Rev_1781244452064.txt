# Naviora by Blue Stratum

## Assessment Engine Refactor

Review the existing Assessment Engine implementation and simplify the domain model.

The platform is a Maritime Simulator Assessment Platform, not a traditional LMS.

Current implementation contains concepts such as:

* Assessment Categories
* Assessment Templates
* Assessment Rules

These introduce unnecessary complexity and should be removed.

Refactor the Assessment domain to a simpler model.

---

# New Assessment Architecture

Exercise Management

* Categories
* Encounter Types
* Exercises
* Exercise Variants

Assessment Management

* Assessments
* Assessment Participants
* Assessment Scheduling
* Assessment Results

---

# Core Principle

An Assessment is simply a collection of Exercises with assessment settings.

Example:

Assessment:

COLREG Level 1 Assessment

Exercises:

* Crossing Assessment
* Head-On Assessment
* Overtaking Assessment

Assessment Settings:

* Duration = 60 Minutes
* Pass Score = 70%
* Attempts = 3
* Certificate Eligible = Yes

No separate Assessment Template entity.

No separate Assessment Rules entity.

Assessment contains everything required.
