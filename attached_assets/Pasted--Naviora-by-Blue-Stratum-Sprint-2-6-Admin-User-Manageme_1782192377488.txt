# Naviora by Blue Stratum

## Sprint 2.6 – Admin User Management, Dynamic RBAC & Organization Ownership

Build the complete Admin User Management and Dynamic RBAC (Role Based Access Control) module.

This sprint establishes the internal administration foundation of Naviora.

Before Organizations, Simulator Runtime, Results and AI modules are implemented, the platform must support:

* Admin User Management
* User Invitations
* Dynamic Roles
* Dynamic Permissions
* Permission Scopes
* Organization Ownership
* Admin Authorization Middleware

This RBAC system must be completely database-driven.

Do NOT hardcode roles.

Do NOT hardcode permissions.

Roles and permissions must be configurable from the UI.

Seed default roles and permissions, but allow modification.

Follow all existing platform standards:

* FastAPI
* PostgreSQL
* SQLAlchemy 2.0
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* JWT Authentication
* Audit Logging
* Encryption Framework
* GDPR Compliance
* Search
* Filters
* Pagination

Generate production-ready code.

* OWNED_ONLY
