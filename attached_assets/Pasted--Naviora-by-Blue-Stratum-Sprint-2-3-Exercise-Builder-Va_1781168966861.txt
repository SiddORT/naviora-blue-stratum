# Naviora by Blue Stratum

## Sprint 2.3 – Exercise Builder & Variant Management

Build the Exercise Builder module for Naviora.

This is a core domain module and must be designed correctly from the beginning.

Important Concept:

An Exercise represents a competency objective.

An Exercise Variant represents an actual simulator setup.

Do NOT duplicate exercises for different weather, visibility, ports or vessels.

One Exercise can contain multiple Variants.

Example:

Exercise:

* Crossing Situation Assessment

Variants:

* Day Crossing
* Night Crossing
* Fog Crossing
* Storm Crossing
* Tanker Crossing
* Multi Vessel Crossing

The platform must support unlimited variants under a single exercise.

Future modules including Assessments, Simulator Sessions, AI Assessment and Reporting will consume Exercise Variants.

Follow all existing platform standards:

* FastAPI
* PostgreSQL
* SQLAlchemy 2.0
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* Audit Logging
* Soft Delete
* Search
* Filters
* Pagination
* Encryption Standards
* GDPR Standards

# Variant Generation Support

Design for future support.

Generation Mode:

MANUAL

Admin manually creates variants.

---

RANDOMIZED

System randomly selects:

* Ports
* Environment Profiles
* Vessels

from allowed pools.

---

PARAMETERIZED

Admin defines constraints.

Example:

Allowed Weather:

* Clear
* Fog
* Rain

Allowed Vessels:

* Container
* Tanker
* Bulk Carrier

System generates scenarios dynamically.

Do not build the generation engine now.

Only create architecture for it.
