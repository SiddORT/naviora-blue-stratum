# Naviora by Blue Stratum

## Sprint 2.2 – Maritime Master Data Hub

Build the Maritime Master Data Hub module.

This module will act as the central source of truth for all maritime master data used throughout the platform.

Future modules including Exercises, Assessments, Simulator Sessions, Incident Scenarios, AI Assessment and Reporting will consume these records.

Follow all existing platform standards:

* FastAPI
* SQLAlchemy 2.0
* PostgreSQL
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* TanStack Query
* React Hook Form
* Zod

Use existing:

* Authentication
* Audit Logging
* Encryption Framework
* DataTable Component
* API Standards
* Storage Architecture
* Soft Delete Standards

# Vessel Library

Purpose:

Store vessel types used by exercises and assessments.

Fields:

* vessel_name
* vessel_code
* vessel_type
* imo_category
* length
* beam
* draft
* max_speed
* maneuverability_rating
* description
* status

Default Vessel Types:

* Container Vessel
* Bulk Carrier
* Oil Tanker
* Chemical Tanker
* LNG Carrier
* Ferry
* Tug
* Fishing Vessel
* Naval Vessel
* Offshore Vessel
* Custom

Search:

* vessel_name
* vessel_code

Filters:

* vessel_type
* status

* Sea State

* weather_condition_id
* sea_state_id
* visibility_condition_id
* time_of_day_id
* description
* status

Example Profiles:

* Clear Day Navigation
* Restricted Visibility Crossing
* Heavy Weather Transit
* Night Port Entry
* Foggy Coastal Navigation
