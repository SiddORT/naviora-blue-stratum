# Naviora by Blue Stratum

## Sprint 2.8B – Candidate Portal Foundation & Basic Proctoring

Build the complete Candidate Portal Foundation and Basic Proctoring Module.

This sprint creates the primary experience for candidates participating in assessments.

Candidates must be able to:

* Login
* View assigned assessments
* Complete assessment check-in
* Review assessment instructions
* Launch assessments
* Track progress
* View session history
* Manage profile
* Complete basic proctoring requirements

This sprint must prepare the platform for:

* Simulator Runtime Engine
* Results Engine
* Certificates
* AI Assessment

Follow platform standards:

* FastAPI
* PostgreSQL
* SQLAlchemy 2.0
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* JWT Authentication
* Audit Logging
* Encryption Framework
* GDPR Compliance
* Search
* Filters
* Pagination

Generate production-ready implementation.

Store all uploaded files through Storage Helper.

Store only file paths in database.

Never store files as BLOBs.

# Basic Proctoring Module

Create:

Purpose:

Track integrity-related events.

Fields:

* id

* assignment_id

* candidate_id

* session_id

* event_type

* event_timestamp

* remarks

Event Types:

* Assessment Started
* Assessment Paused
* Assessment Resumed
* Assessment Completed
* Browser Lost Focus
* Browser Regained Focus
* Session Timeout
* Candidate Disconnected

Purpose:

Monitoring only.

Do not block assessments.
