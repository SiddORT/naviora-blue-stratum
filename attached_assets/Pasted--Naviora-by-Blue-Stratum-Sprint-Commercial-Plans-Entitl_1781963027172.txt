# Naviora by Blue Stratum

## Sprint – Commercial Plans & Entitlements

Build the Plans & Entitlements module.

This module manages commercial subscriptions, feature access and content licensing for organizations.

The purpose is to dynamically control:

* Features
* Simulator Access
* Exercise Access
* Assessment Limits
* User Limits
* Storage Limits
* Offline Access
* AI Features

Follow existing platform standards:

* FastAPI
* PostgreSQL
* SQLAlchemy 2.0
* Alembic
* Pydantic v2
* Next.js 15
* TypeScript
* Audit Logging
* Soft Delete
* Search
* Filters
* Pagination

Generate production-ready code.

Do not use hardcoded plan logic.

All plan behavior must be database driven.

---

# Business Concept

A Plan defines what an Organization can access.

Example:

Starter Plan

* 50 Candidates
* 10 Assessments per Month
* COLREG Exercises Only

Enterprise Plan

* Unlimited Candidates
* Unlimited Assessments
* All Exercises
* AI Reporting

Organizations subscribe to Plans.

Plans can be changed without code changes.
